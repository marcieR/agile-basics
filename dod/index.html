Quality Gates

Definition of Done
& Definition of Ready

Two agreements that eliminate the most common source of sprint failure — ambiguity about when work starts and when it ends.

✅

Definition of Done

A shared, team-owned checklist that every story must pass before it can be called complete. Not "done enough" — done.

Why it matters

Without a DoD, "done" means something different to every team member — engineers think done means code merged, QA thinks done means tested, PO thinks done means in production. The DoD makes the implicit explicit. It replaces subjective judgment with a shared, enforceable standard that the whole team has agreed to uphold, sprint after sprint.

The checklist
Code Quality
Code reviewed and approved
No new linting errors
Code merged to main branch
Testing
All AC tested and passing
Unit / integration tests written
CI pipeline passes
Tested on staging / target environment
Delivery
Deployed to staging or production
PO has reviewed and accepted
Ticket updated in project tool
Documentation (where applicable)
Relevant docs updated
Release notes / changelog updated
💡
How to build yours

Run a session in Sprint 0 or a retro. Ask: "what would have to be true for us to confidently call a story done?" Debate until everyone agrees. Enforce it every sprint. Revise quarterly.

🔷

Definition of Ready

The criteria a story must meet before the team pulls it into a sprint. If it's not ready, it doesn't get planned.

Why it matters

Prevents mid-sprint chaos from stories that entered planning unclear. A story that isn't ready doesn't belong in a sprint — it belongs back in refinement. The DoR protects the team's ability to commit confidently and deliver without constant interruption for missing information.

The checklist
Clarity
User story in standard format (As a / I want / So that)
AC written and testable
The "why" is understood
Scope & Size
Completable in one sprint
Estimated by the team
Edge cases and error states defined
Dependencies
Dependencies identified and resolved or flagged
Design assets available if applicable
Three Amigos review completed
DoD vs DoR — the key difference
Definition of Done

End of the story. Owned by the whole team. Answers: "Can we ship this?" Applied when work is complete.

Definition of Ready

Start of the story. Owned by the Product Owner. Answers: "Can we start this?" Applied before sprint planning.

Build them as a team
1

Run a team session — Sprint 0, a retro, or a dedicated working session. Everyone present.

2

Start minimal — Aim for 5–8 items each. A checklist nobody enforces is just decoration.

3

Post them visibly — In your project tool, on a wiki page, or a physical board. If it's not visible, it's not used.

4

Revisit every quarter — As the team matures, the bar rises. Your DoD and DoR should too.
